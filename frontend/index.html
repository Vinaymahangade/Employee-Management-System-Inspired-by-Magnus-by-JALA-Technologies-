What is JWT Authentication?▼

JWT (JSON Web Token) is a compact, URL-safe token format used for securely transmitting information between parties. In our system, on login the server generates a signed JWT containing the user's email and role. The client stores it and sends it in every subsequent request via the Authorization: Bearer <token> header.

What is Soft Delete?▼

Soft delete means the record is not physically removed from the database. Instead, its status is set to INACTIVE. This preserves historical data, allows audit trails, and lets you reactivate records. Our DELETE /api/employees/{id} endpoint performs a soft delete.

How does Role-Based Access work?▼

Roles are stored in the employees table (ADMIN or USER). Spring Security reads the role from the JWT, and endpoints are protected using @PreAuthorize("hasRole('ADMIN')"). The UI also hides admin-only buttons for USER-role accounts.

How is pagination implemented?▼

The search endpoint accepts page, size, sortBy, and sortDir query parameters. Spring Data JPA's Pageable interface handles the SQL LIMIT, OFFSET, and ORDER BY automatically. The response includes totalElements, totalPages, and content array.

Tech Stack Details▼
  • Backend: Spring Boot 3.2, Spring Security, Spring Data JPA
  • Database: PostgreSQL 15 with Hibernate ORM
  • Auth: JWT (JJWT library), BCrypt password hashing
  • Docs: Swagger UI via SpringDoc OpenAPI
  • Frontend: Vanilla HTML, CSS, JavaScript (no frameworks)
Autocomplete Search

Type at least 2 characters to see live suggestions from the backend API.

How it works

  1. User types in the input field
  2. Debounced API call fires after 300ms pause: GET /api/employees/autocomplete?query=john
  3. Backend returns top 10 matching employees by name or email
  4. Results are rendered as a dropdown list
  5. Clicking a result selects it and closes the dropdown
